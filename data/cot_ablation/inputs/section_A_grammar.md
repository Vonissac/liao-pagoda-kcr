# 辽金砖石塔建筑结构描述语言 LPSL v3.0 规范 (Section A )

## 1. 核心营建符号与严格边界铁律

本语言通过一维字符串序列描述古建筑的垂直与水平空间。模型必须严格遵守以下符号组合铁律：

### 1.1 起始与结束边界
* 所有编码必须以 `Global(Sides=X,BaseR=Y)+a` 严格开头。其中 `X` 为塔的平面边数（如 8 或 4），`Y` 为基座半径（浮点数）。
* 全序列必须以 `-i`（若有前刹则为 `-i-l`）绝对闭合。

### 1.2 属性槽 `[...]` 的严格位置限制
* **铁律**：属性槽 `[...]` **只能且必须**紧跟在特定构件代码（如铺作层 `d`、壸门层 `g`、塔身 `e`）的后面，用于声明其特定样式、类型或数量。**严禁在其他无属性构件（如 `c`, `b`, `h`, `i` 等）后挂载中括号。**

### 1.3 嵌套算子 `(...)` 与内部空间切分符号
嵌套算子 `(...)` 用于表达复合构件（如塔身 `e`、壸门 `g`）的内部单面墙体装饰。其内部空间切分符号定义如下：
* **水平次序算子 `-`**：在 `(...)` 内部，`-` 代表**同一层墙面上的水平并列平铺次序**（从左向右）。
* **垂直分层算子 `/`**：在 `(...)` 内部，`/` 代表**单面墙体上的垂直向下分层**（从上到下）。从左向右每出现一个 `/`，代表墙面视觉上下移一层。

## 2. 顶级构件序列白名单 (Whitelist)
顶级构件之间通过 `-` 从左向右连接（物理上对应从塔顶到地表）。**严禁输出此列表以外的任何顶级 ID。**

| 代码 | 名称 | 语法挂载约束与物理意义 |
| :--- | :--- | :--- |
| **a** | 顶层特制塔檐 | 必须且只能出现在全序列的最前端（紧跟在 Global 后面）。 |
| **c** | 标准塔檐 | 檐区的核心单元，后方严禁挂载 `[...]`。 |
| **b** | 叠涩层/束腰 | 衔接与过渡层。 |
| **r** | 反叠涩层 | 上窄下宽的过渡层。 |
| **d** | 铺作层/斗栱层 | 檐下支撑。**必须紧跟属性槽**，如 `d[type=d1]`。 |
| **m** | 平座层 | 楼阁式平座。 |
| **h** | 素面层 | 物理平铺过渡层。 |
| **e** | 塔身核心 | 全塔分界点。**必须挂载属性与嵌套**，形如 `e[col=eX](内部装饰)`。序列中最后一个 `e` 左侧为檐区，右侧为基区。 |
| **f** | 莲花层 | 仰莲或覆莲座，通常在塔身下方。 |
| **g** | 壸门层 | 塔基核心层。**必须挂载属性与嵌套**，形如 `g[num=X](内部装饰)`。 |
| **k** | 栏板 | 塔基防护/装饰栏板。 |
| **o** | 勾栏 | 塔基防护/装饰勾栏。 |
| **j** | 雕像层 | 独立的物理雕像层。 |
| **n** | 塔基假门 | 塔基部分的雕刻假门。 |
| **i** | 塔基最底层 | 物理地表层，全塔的绝对终点。 |
| **l** | 前刹 | 塔前独立结构，若存在，则全序列以 `-i-l` 结尾。 |

## 3. 嵌套装饰层与细部构件代码白名单
当且仅当在复合构件的嵌套括号 `(...)` 内部时，允许使用以下细部装饰代码进行 `-` 或 `/` 的拼装。

### 3.1 塔身嵌套 `e[col=eX](...)` 内部可用代码
* **转角柱样式白名单（仅用于属性槽 `col=`）**：`e1` (普通柱), `e2` (八角柱), `e3` (圆柱), `e4` (束竹柱), `e5` (梅花柱)。
* **墙面装饰构件代码白名单（仅用于圆括号 `(...)` 内部）**：
  * `e6` : 破子棂窗
  * `e7` : 格扇窗
  * `e8` : 壸门板
  * `e9` : 倚柱/间柱
  * `e10` : 砖雕帷幕/华盖
  * `e11` : 雕刻假门（双扇开）
  * `e12` : 雕刻假窗
  * `e13` : 飞天雕刻
  * `e14` : 佛龛
  * `e15` : 菩萨/神将雕像
  * `e16` : 砖砌假窗（闭合）
  * `e17` : 纯素面墙体

### 3.2 壸门层嵌套 `g[num=X](...)` 内部可用代码
* **数量属性（仅用于属性槽 `num=`）**：`X` 必须为具体正整数（如 `1`, `2`, `4`）。
* **内部单元代码白名单（仅用于圆括号 `(...)` 内部）**：
  * 间柱部分直接复用：`e1` 到 `e5` 柱体代码。
  * `g8` : 壸门内雕狮子
  * `g9` : 壸门内雕乐伎
  * `g10` : 壸门内雕植物花卉
  * `g11` : 壸门内雕化生童子
  * `g12` : 空壸门（无雕刻）

## 4. 严格生成准则
1. 必须根据历史文献描述的实际物理结构进行转换。
2. 属性槽 `[...]` 严禁滥用，只能出现在 `Global`、`d`、`e`、`g` 构件代码的后方。
3. 圆括号 `(...)` 内部的横轴横线 `-` 和纵轴斜线 `/` 必须符合逻辑，且所有括号必须成对闭合。
4. **禁止任何解释**：模型仅允许输出一行最终推导出的标准 LPSL v3.0 字符串。
