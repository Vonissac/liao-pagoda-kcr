# 辽金砖石塔建筑结构描述语言 LPSL v3.0 知识底座规约 (Section B 静态知识增强专用)
所有编码必须以 `Global(Sides=X,BaseR=Y)+a` 严格开头。其中 `X` 为塔的平面边数（如 8 或 4），`Y` 为基座半径（浮点数）。
* 全序列必须以 `-i`（若有前刹则为 `-i-l`）绝对闭合。

## 1. 空间映射核心轴向与算子流体系 (Knowledge Map)
本规范作为辽金砖石塔的结构知识底座，定义了三维建筑空间向一维符号序列映射的完备算子流：
1. **整体物理方向（自左向右 ≡ 自上至下）**：编码从左向右书写，物理上严格对应从塔顶（上部塔檐）逐渐过渡到塔底（下部塔基）。
2. **水平次序算子 (`-`)**：
   * 在顶级构件间，`-` 代表不同物理结构层自上而下的顺次连接。
   * 在嵌套复合构件 `(...)` 内部，`-` 代表同一层墙面上的水平并列平铺次序（空间上自左向右）。
3. **垂直分层算子 (`/`)**：
   * 仅在复合构件嵌套 `(...)` 内部使用，代表单面墙体上装饰构件的垂直向下切分。自左向右每出现一个 `/`，代表视觉上下移一层。

---

## 2. 顶级构件序列完备白名单 (Structural Layer Whitelist)
顶级构件之间通过 `-` 连接。最后一个出现的 `e` 构件是全塔的核心空间分界点（`e` 左侧严格为塔檐区，`e` 本身为塔身，`e` 右侧严格为塔基区）。

| 代码 | 名称 | 允许出现的空间段 | 语法约束、物理意义与典型位置 |
| :--- | :--- | :--- | :--- |
| **a** | 顶层特制塔檐 | 塔檐区（最左侧） | **必须**出现在序列最前端（顶层）。因其形式与标准塔檐不同，故独立编码。 |
| **c** | 标准塔檐 | 塔檐区 | 密檐塔的核心重复单元。 部分层数较低的塔无塔檐，以铺作计层|
| **b** | 叠涩层/束腰 | 塔檐区 / 塔基区 | 衔接与过渡层。在塔檐部分常与 $c$ 或 $h$ 形成循环；在塔基部分作为束腰或衔接。 |
| **r** | 反叠涩层 | 塔檐区 / 塔基区 | 衔接过渡。特征是上层宽度小于下层宽度（上窄下宽）。在塔檐部分可替代 $c$ 和 $b$ 一同形成循环。 |
| **d** | 铺作层/斗栱层 | 塔檐区 / 塔基区 | 檐下支撑结构。常与 $c$ 结合或在檐层附近出现，是密檐塔塔檐部分的核心重复单元。通常带有属性如 `[type=d1]`。 |
| **m** | 平座层 | 檐区 / 塔身连接处 / 塔基 | 楼阁式塔极其常用，可出现在塔身上与塔檐连接处、塔檐部分，亦可出现在塔基。 |
| **h** | 素面层 | 全区（檐/身/基） | 物理支撑与过渡平铺层，各部分均有极高概率出现。常用于辅助支撑平座。 |
| **e** | 塔身 | 塔身区（核心分界点）| 承载主要装饰。**序列中最后一个出现的 e 即为塔身**。 |
| **f** | 莲花层 | 塔基区（偏上） | 装饰性过渡层。通常出现在塔身下方、塔基的最上层，也可能出现在塔基的其他过渡位置。 |
| **g** | 壸门层 | 塔基区 | 通常出现在塔基部分。必须携带数量属性与内部拓扑，如 `g[num=2](e1-g10)`。**严禁连续重复超过 3 次**。 |
| **k** | 栏板 | 塔基区 / 塔檐区 | 装饰性结构层，出现在塔基或塔檐部分，以塔基为主，位于外侧。 |
| **o** | 勾栏 | 塔基区 / 塔檐区 | 装饰性结构层，出现在塔基或塔檐部分，以塔基为主，位于外侧。 |
| **j** | 雕像层 | 塔基区 / 塔身区 | 独立的物理雕像层。 |
| **n** | 塔基假门 | 塔基区（末尾） | 塔基部分的特殊结构，仅极少量塔有，供人进入。若出现，必须编码在最后、i 之前，以 `n-i` 或 `n-i-l` 形式结尾。 |
| **i** | 塔基最底层 | 塔基区（绝对终点） | 物理地表层/最底层假门。如果没有前刹 $l$，则 $i$ 必在最右侧作为整个 LPSL 的收尾。 |
| **l** | 前刹 | 塔基区（独立终点） | 位于塔前的独立结构（类似前殿）。由于顶部塔刹大多不存固不参与编码。若出现，必须编码在 $i$ 之后，以 `i-l` 绝对结束。 |

## 3. 嵌套装饰层与细部构件完备字典

### 3.1 塔身嵌套 `e[col=eX](...)` 内部可用代码
* **转角柱样式（属性槽 `col=`）**：`e1` (普通柱), `e2` (八角柱), `e3` (圆柱), `e4` (束竹柱), `e5` (梅花柱)。
* **墙面装饰构件代码（圆括号 `(...)` 内部）**：
  * `e6`: 破子棂窗 | `e7`: 格扇窗 | `e8`: 壸门板 | `e9`: 倚柱/间柱 | `e10`: 砖雕帷幕/华盖 
  * `e11`: 雕刻假门（双扇开） | `e12`: 雕刻假窗 | `e13`: 飞天雕刻 | `e14`: 佛龛 
  * `e15`: 菩萨/神将雕像 | `e16`: 砖砌假窗（闭合） | `e17`: 纯素面墙体

### 3.2 门层嵌套 `g[num=X](...)` 内部可用代码
* **数量属性（属性槽 `num=`）**：`X` 必须为具体正整数（如 `1`, `2`, `4`）。
* **内部单元代码（圆括号 `(...)` 内部）**：
  * 间柱部分直接复用：`e1` 到 `e5` 柱体代码。
  * `g8`: 壸门内雕狮子 | `g9`: 门内雕乐伎 | `g10`: 门内雕植物花卉 | `g11`: 门内雕化生童子 | `g12`: 空壸门（无雕刻）

## 4. 径向多面体循环表达规范
当 `Sides` 存在时，若在顶级构件后出现多组并列的 `(...)`，代表塔的各面在径向绕塔一周时交替不同：
* 单组括号 `(A)`：全塔所有墙面完全一致，该结构连续重复 `Sides` 次。
* 两组括号 `(A)(B)`：$A$ 样式面与 $B$ 样式面交替出现。全塔循环 `Sides / 2` 次（如假门面与假窗面交替）。
* 密檐塔层数大多为奇数且层数较多，楼阁塔则较少，可能出现偶数层。密檐塔层数等于a与c出现之和，楼阁塔为e的出现次数

## 5. Few-shot 静态标准长编码形态示例 (无推理过程)
*注意：以下示例仅用于展示基于本知识底座生成的最终标准 LPSL v3.0 字符串形态，作为格式对照基准。*

* **输入描述 1**：8角砖塔，基座半径8.5米。共13层檐。顶层檐特殊。往后是12层密檐循环，最后一层檐下有斗栱。塔身转角为普通柱，墙面最上层为飞天-莲台-飞天；中间层为三个华盖；底层为人物像-假门-人物像。塔身下方是莲花座，接着是塔基壸门层，每面有2个壸门，门间柱为普通柱，门内雕刻为植物。壸门外侧带有双层栏板，最后过渡到最底层地表。
* **标准 LPSL 编码 1**：`Global(Sides=8,BaseR=8.5)+a-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-b-h-c-d[type=d1]-e[col=e1](e6-e14-e6/e9-e9-e9/e10-e8-e10)-f-g[num=2](e1-g10)-k-k-i`

* **输入描述 2**：六角砖塔，基座半径6米，共3层檐。第一层顶檐带平座斗栱；第二层檐由反叠涩替代标准檐；第三层采用标准塔檐。塔身采用龙柱转角，且六个墙面中，假门面与假窗面交替出现。塔基有莲花层、平座层、门层（每面3个，无间柱，内雕人物），供人进入的假门，底层地面，及独立前刹。
* **标准 LPSL 编码 2**：`Global(Sides=6,BaseR=6.0)+a-m-d-h-r-m-d-h-c-m-d-h-e[col=e2](e8)(e11)-f-m-g[num=3](e5-g8)-n-i-l`

**输入描述 3**：六角砖塔，基座半径6.0米。共3层塔身与塔檐顺次交替。第一层（顶层）顶檐特殊，且檐下带有平座与斗栱；第二层檐物理上发生异变，由反叠涩（上窄下宽）替代标准塔檐，檐下同样带平座斗栱；第三层（底层）采用标准塔檐与檐下平座斗栱。塔身转角采用龙柱，且六个墙面中，假门面与假窗面顺时针交替出现。塔基从上到下依次有莲花层、平座层、壸门层（每面3个壸门，无间柱，内雕人物），随后为通行假门、底层地面，并在最外侧设有独立的物理前刹结构。
* **标准 LPSL 编码 2**：`Global(Sides=8,BaseR=5.0)+a-m-d[type=d1]-e[col=e1](e8)(e11)-o-d[type=d1]-c-m-d[type=d1]-e[col=e1](e8-e10)(e11)-o-d[type=d1]-c-m-d[type=d1]-e[col=e1](e8)(e11)-o-d[type=d1]-c-m-d[type=d1]-e[col=e1](e8)(e11)-o-d[type=d1]-c-m-d[type=d1]-e[col=e1](e8)(e11)-i`

## 6. 生成准则
1. 严格基于本知识底座提供的规则与构件白名单，直接比对输入的历史文献映射出对应的结构编码。
2. 圆括号 `(...)` 内部的横轴横线 `-` 和纵轴斜线 `/` 必须符合空间划分，且所有括号必须完全成对闭合。
3. **禁止输出任何形式的思考过程、分步推理、校验逻辑或中间文本。** 模型必须且只能输出一行最终的、标准的 LPSL v3.0 字符串。
